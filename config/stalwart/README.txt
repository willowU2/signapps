Stalwart Mail Server — Configuration
======================================

Stalwart stores its configuration inside the Docker volume
(signapps-stalwart-data → /opt/stalwart-mail).

On first run Stalwart auto-generates its configuration.
You can customise it via the management API at http://localhost:8580
or by editing the TOML files inside the volume.

Recommended dev setup:
  - Domain       : signapps.local
  - TLS          : self-signed (auto-generated by Stalwart)
  - IMAP port    : 993 (TLS) or 143 (plain, requires STARTTLS config)
  - SMTP port    : 587 (submission) or 25 (relay)
  - Admin API    : http://localhost:8580

The startup script is at: scripts/start-stalwart.sh

Environment variables used by signapps-mail for internal server:
  STALWART_API_URL    — Management API base URL (default: http://localhost:8580)
  STALWART_API_TOKEN  — Admin bearer token for Stalwart management API
  STALWART_IMAP_HOST  — IMAP host for internal accounts (default: localhost)
  STALWART_IMAP_PORT  — IMAP port for internal accounts (default: 993)
  STALWART_SMTP_HOST  — SMTP host for internal accounts (default: localhost)
  STALWART_SMTP_PORT  — SMTP port for internal accounts (default: 587)
