SignApps Project - Keep UI Redesign (Google Keep Style)

You are the Autonomous AI Development Team for "SignApps". Your task is to implement the "Keep App Redesign" using the existing Next.js / Tailwind stack.

**Context & Requirements:**
1. Build or refactor the Keep interface (`client/src/app/keep/page.tsx` or similar) to use a dark mode masonry layout.
2. At the top of the interface, create a central search bar ("Recherche").
3. Below the search bar, implement the "Créer une note..." input bar (which expands when clicked).
4. Implement a masonry grid layout for the note cards.
5. The cards must support different label categories (chips at the bottom), list checklists with strikethrough logic, and thin, subtle borders matching the classic Google Keep dark mode visual style.

**Aesthetics:**
Ensure the masonry grid calculates gaps perfectly. Note backgrounds should be pure dark gray/black depending on the theme, with very subtle borders. 

**Constraints:**
1. Use existing data logic if present, or create the necessary Zustand store to manage note items.
2. Use existing libraries only (`lucide-react`, `tailwindcss`, `shadcn/ui`, and existing masonry utilities if you have them, otherwise use CSS columns).
3. Validate compilation via `npm run lint` and `npx tsc --noEmit`.
