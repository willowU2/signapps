### Module mail (2026-04-16)

Reviewer: feature-dev:code-reviewer subagent.
Findings: P0=2, P1=4, P2=3

**Fixed:**
- **P0** `client/src/components/mail/mail-display.tsx:1055` — iframe sandbox was
  `allow-same-origin`, granting untrusted HTML email full access to parent
  localStorage/cookies. Script-strip regex was trivially bypassable. Changed
  to `sandbox=""` (fully sandboxed).
- **P0** `client/src/app/mail/templates/page.tsx:540` — raw HTML injection
  received server-stored template HTML unsanitized. Routed through DOMPurify
  via new `TemplatePreviewHtml` helper component.
- **P1** `client/src/components/mail/mail-display.tsx:845,1036` — PGP components
  received `mail.id` (message UUID) as `accountId` instead of `mail.account_id`.
  PGP key lookup permanently returned "no key" even when one existed. Fixed to
  `mail.account_id ?? accountId ?? ""`.
- **P1** `services/signapps-mail/src/handlers/email_send.rs:59` — added CRLF
  rejection on recipient/subject/cc/bcc to block SMTP header injection.
- **P1** `services/signapps-mail/src/sync_service.rs:33` — `.expect("Failed to
  build TLS connector")` violated the zero-unwrap rule. Refactored to return
  `Result<TlsConnector, native_tls::Error>`. Updated single caller in
  accounts_connection.rs to propagate.
- **P1** `client/src/components/mail/mail-display.tsx:326` — replyAll silently
  dropped cc recipients. Added TODO comment explaining the Mail type gap
  (mail.cc / mail.recipients fields absent). No behavior change (already was
  equivalent to reply); documentation now honest.
- **P2** Removed 4× `console.warn`/`console.error` in mail-display/pgp-settings/
  pgp-indicator (replaced by existing `toast.error` calls, no info loss).

**Skipped (not bugs):**
- Mail type `mime_type` vs `mimeType` duplication — consolidation touches 30+
  files across storage/drive/mail/docs. Deferred to a dedicated cleanup task.
