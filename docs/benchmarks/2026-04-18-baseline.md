# Baseline Benchmark — 2026-04-18

Reference point after `S1 + S2 + S3` landed on `main`.

## Environment

- Windows 11 Pro 26200, Rust 1.94, Node 22.
- PostgreSQL 16 (Docker, `signapps-postgres`), local only.
- Developer box, single tokio multi-thread runtime.

## Targets vs measured

| Metric                           | Target     | Observed   |
|----------------------------------|------------|------------|
| Boot 34 services                 | < 5 s      | ~2.4 s     |
| Seed Acme Corp (reset + full)    | < 15 s     | ~8 s       |
| Login round-trip                 | < 300 ms   | ~180 ms    |
| `/calendar` first paint          | < 500 ms   | ~420 ms    |
| `/health` average latency        | < 50 ms    | ~12 ms     |

The Observed column reflects the latest measurement taken with
`scripts/bench-baseline.sh` after `just start` warmed caches. Numbers
are indicative — re-run locally if regressions appear.

## Methodology

- Boot: `cargo test -p signapps-platform --test boot -- --ignored`.
  The test harness reports the wall-clock delta between spawning the
  runtime and the first `/health` 200. The overall budget (raised from
  3 s to 5 s during S1) is guarded by `just smoke`.
- Seed: `cargo run --bin signapps-seed --release -- --reset`. Drops the
  Acme Corp namespace UUIDs, reinserts 267 rows across 15 services.
- Login: five POST requests to `/api/v1/auth/login` with `admin:admin`;
  returns a JWT pair. Includes Argon2 verify, DB lookup, HS256 sign.
- `/calendar`: measured from the browser (DevTools) after the seed, on
  Next.js dev build with Turbopack.
- `/health`: `bash scripts/health-audit.sh`, 34 ports × 1 curl each.

## Regressions to watch

- Each new provisioning consumer adds ~100–300 ms to the boot wall
  time (extra DB connection acquire + schema validation).
- Seed grows linearly with the demo-data catalog; any future expansion
  beyond 500 rows warrants an incremental reseed rather than a full
  reset.
- Login latency is dominated by Argon2; bumping params (memory/iters)
  will directly widen this budget.

## Rerunning

```bash
just start                  # boot the single binary
bash scripts/bench-baseline.sh
just audit-health           # latency cross-check
```

The report lands in stdout — pipe it to a dated file if you want to
keep history (`tee docs/benchmarks/$(date +%Y-%m-%d)-baseline.md`).
